## 1. Executive Summary

6개월간 실장님들과 플랫폼서비스실 간의 카톡 단톡방을 정량·정성 분석한 결과,
**(a) 자주 반복되는 문의가 명확한 패턴을 가지고 있고, (b) 그 답변은 이미 누적되어 있으나 (c) 검색 불가능한 휘발성 텍스트에 묻혀 있다**는 점을 확인했습니다.

본 분석에서 카톡 단톡방을 다음 세 가지 위키 자산으로 변환했습니다:

| 산출물 | 위치 | 효용 |
|---|---|---|
| **신규 가이드 7개** | `src/data/inquiryGuides.js` → `mockData.js` 통합 | 카톡에서 가장 많이 인입된 미커버 영역을 위키 가이드로 표준화 |
| **분석 데이터셋** | `src/data/inquiryAnalytics.js` | 지점·카테고리·시계열·FAQ Top 25·미해결 백로그를 구조화 |
| **검색 동의어 확장** | `mockData.js`의 `SEARCH_SYNONYMS` 6→14 키워드 | 자연어 문의를 가이드로 라우팅 |

기대 효과 (정량 추정):
- 플서실 응대 부하 **약 30~50% 감소** (반복 문의 위키 자가 해결)
- 실장 대기시간 **약 70% 감소** (즉시 검색 가능)
- 신규 입사자 온보딩 **2주 단축** (실전 시나리오 누적)

## 6. 추가된 위키 신규 가이드 7개

위 패턴을 표준 절차로 변환하여 위키 가이드로 추가했습니다 (`src/data/inquiryGuides.js`):

| ID | 유형 | 제목 | 카톡 발생 | 비고 |
|---|---|---|---:|---|
| `refund-pending-cancel` | TROUBLE | 환불대기 취소 처리 (수작업 요청 절차) | 280회 | 1순위 요청 |
| `okta-device-reset` | SOP | OKTA 인증 기기변경 / 재설정 절차 | 42회 | 월 5~8건 |
| `shinhan-campus-payment` | DECISION | 신한캠퍼스 결제·환불 처리 결정 가이드 | 58회 | 정책 혼선 최다 |
| `cti-duplicate-payment` | TROUBLE | CTI 콜 인입 중 중복결제 처리 | 18회 | 5월 기준 미해결 |
| `parent-phone-change` | SOP | 학부모 대표번호 변경 (부↔모) | 50회 | 회원관리 자주 |
| `payment-url-expired` | TROUBLE | 결제 URL 만료 / 미수신 트러블슈팅 | 35회 | 6가지 원인 분기 |
| `live-data-isolation` | RESPONSE | 라이브(sdijon) 수강 데이터 이관 불가 안내 | 22회 | CS 응대 스크립트 |

각 가이드는 다음 메타데이터를 포함합니다:
- `inquirySource`: 카톡 발생 빈도·대표 지점·샘플 문의 3건
- `decisionTable` / `troubleTable`: 케이스별 분기
- `cautions`: 정책상 주의사항
- `tags`: 검색 식별자 (`자주묻는질문`, `실장`, `필수` 등)

## 8. 추후 작업 제안 (Roadmap)

### Tier 1 — 단기 (1~2개월)
1. **자가 검색 위키화** (✅ 본 PR로 완료): 신규 가이드 7개 + 분석 데이터셋 통합
2. **실장용 챗봇 (RAG)**: 위 가이드를 임베딩 후 AMS 내 위젯에서 자연어 검색
3. **카톡 자동 응답봇**: 새 질문 인입 시 위키 유사 가이드 자동 인용

### Tier 2 — 중기 (2~3개월)
4. **이슈 트래커 동기화**: 카톡의 "확인부탁" 키워드 감지 → Jira 자동 티켓 생성
5. **지점별 페르소나 대시보드**: `inquiryAnalytics.js` 기반 시각화
6. **반복 이슈 알람**: 카테고리당 인입 임계치 초과 시 슬랙 알람

### Tier 3 — 장기 (3~6개월)
7. **개인정보 마스킹 파이프라인**: 카톡 데이터 자동 익명화 (학생명·번호·이메일·승인번호)
8. **신규 실장 온보딩 큐레이션**: 지점·역할별 Top 30 FAQ 자동 제공
9. **공지/배포 노티스 타임라인**: 플서실 공지 메시지를 버전 히스토리로 정리
10. **사용자 음성 → 제품 백로그**: "~~기능 있으면 좋겠습니다" 패턴 자동 수집

## 10. 부록 — 분석 메소드

1. **데이터 추출**: 카카오톡 단톡방 export 파일 (txt, UTF-8, 1.1MB)
2. **파싱**: 라인 단위 `[발신자] [시간] 메시지` 패턴 그레핑
3. **카테고리 분류**: 키워드 사전 기반 빈도 카운팅 (예: 환불대기, 통합회원, OKTA)
4. **시계열 분류**: `--------------- YYYY년 M월 D일 ---------------` 헤더 기반 일별 그룹
5. **표준답변 추출**: 플서실 멤버(@플랫폼서비스실 *) 응답을 케이스별로 정리
6. **위키 변환**: SOP/TROUBLE/DECISION/RESPONSE 6유형으로 매핑

> **익명화 정책**: 본 분석에 등장한 학생명·전화번호·이메일·승인번호는 모두 위키에 포함되지 않으며, 가이드의 `inquirySource.sampleRequests`는 패턴만 보존하고 개인정보는 제거되어 있습니다.
