## 0. 경영 요약 (Executive Summary)

3차례의 개발 라운드를 거쳐 AMS Wiki는 **콘텐츠 뷰어 + 검색 + 피드백 + 기본 RBAC** 수준의 MVP 를 달성했다. 디자인 시스템(shadcn/ui)과 코드 스플리팅·타입 센터라이제이션 등 기반 품질은 양호하나, **운영 투입 전 반드시 해소해야 할 보안·권한 관문 2건과 어드민 페이지 부재**가 최대 리스크로 확인됨.

| 영역 | 현 수준 | 판정 |
|---|---|---|
| 사용자 뷰어 (홈/가이드/검색/FAQ/업데이트) | 기능 완성 · DS 준수 · 모바일 대응 | ✅ Production Ready |
| 편집기 (/editor) | 작성 UI 존재 · DB 저장 경로 미비 | ⚠️ MVP 미완 |
| 어드민 (/admin) | **부재** | ❌ 신규 구축 필요 |
| 인증·권한 (RBAC) | user_metadata 기반 · 라우트 가드 없음 | ⚠️ 보안 취약 |
| Confluence 동기화 | 스크립트 준비 완료 · 권한 대기 | ⏸ 외부 블로커 |
| 테스트 커버리지 | 1 파일 (4 테스트) | ❌ 심각 부족 |

**P0 (배포 전 필수)** 3건, **P1 (2주 내)** 9건, **P2 (점진 개선)** 11건이 식별됨.

## 4. 🏗️ 어드민 설계 검수 결과

### 4.1 현 상황

| 영역 | 현재 | 결함 |
|---|---|---|
| 편집기 `/editor` | UI 존재 · autosave (localStorage) | DB 저장·재로드 불가 |
| 가이드 목록 `/admin/guides` | **없음** | 비발행/대기/검토 상태 구분 없음 |
| 피드백 수신함 `/admin/feedback` | **없음** | localStorage 큐 + `guide_feedback` 테이블 병합 화면 없음 |
| 대시보드 `/admin` | **없음** | `fetchDashboardStats` 있으나 호출처 없음 |
| 사용자 관리 `/admin/users` | **없음** | V2 로 이월 |

### 4.2 MVP 범위 (즉시 구축)

1. **AdminLayout** — 좌측 admin-sidebar (app-sidebar 와 분리) · 탑바 user avatar · 파란 강조 theme
2. **RequireRole** 라우트 가드 — `permission="edit"` 또는 `role="admin"` 통과 못하면 /403 리다이렉트
3. **`/admin`** — 대시보드 (기존 `fetchDashboardStats` 재사용)
4. **`/admin/guides`** — 상태 필터 (draft/published/archived) + 발행/해제/삭제 액션
5. **`/editor?id=…`** — 기존 가이드 로드 + `upsert` 저장 (현재 가장 큰 결함)
6. **`/admin/feedback`** — 로컬 큐 + Supabase `guide_feedback` 머지 뷰

### 4.3 V2 (이월)

- 사용자·역할 관리 (`/admin/users`)
- Confluence 수동 동기화 버튼 (`/admin/sync`)
- 변경 이력/Audit log (`/admin/audit`)
- 가이드 요청 접수함 (`guide_requests` 테이블 신규)
