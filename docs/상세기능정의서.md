## 0. 개요

본 문서는 AMS Wiki 의 **기능별 상세 사양(Behavior · Data · Error)** 을 개발·QA·운영이 단일 출처로 참조할 수 있도록 정리한 문서이다. 엔드유저 뷰어·에디터·어드민 3개 축을 모두 포함한다.

### 역할 매트릭스

## 1. 인증 · 세션

### 1.1 로그인 (이메일/비밀번호)
- **Trigger**: `UserMenu` 드롭다운 내 로그인 폼 submit
- **Input**: `email` (string, required, email format) · `password` (string, required, ≥ 6)
- **Flow**:
  1. `supabase.auth.signInWithPassword()` 호출
  2. 성공 시 `onAuthStateChange` 가 user state 갱신
  3. 실패 시 `<Toast variant="destructive">` 로 에러 노출
- **Fallback**: `isSupabaseEnabled === false` → `fallbackLogin(email)` 로 OPERATOR 역할 부여 (데모)
- **Output**: 상단 avatar + 한글 역할 라벨 노출

### 1.2 회원가입
- **Input**: `email`, `password`, `name`
- **Output**: Supabase `signUp` → email verification 링크 전송 (운영 환경)
- **기본 역할**: `OPERATOR` (관리자가 직후 승격 가능)

### 1.3 로그아웃
- `supabase.auth.signOut()` + `localStorage.removeItem('ams_wiki_user')`

### 1.4 세션 지속
- Supabase 자동 refresh (기본 60분 만료 → silent refresh)
- `AuthProvider` 는 `getSession()` 로 첫 마운트 동기화, 이후 `onAuthStateChange` 구독

### 1.5 RBAC 판정 함수

| 함수 | 시그니처 | 용도 |
|---|---|---|
| `hasPermission(perm)` | `('view'\|'edit'\|'publish'\|'delete'\|'manage_users') → boolean` | 버튼·액션 게이트 |
| `hasRole(role)` | `(role) → boolean` | 특정 역할 체크 |
| `canAccess(moduleId)` | `(moduleId) → boolean` | 사이드바·라우트 노출 |

## 2. 엔드유저 페이지

### 2.1 홈 `/`
- **데이터**: `useRecentGuides(8)`, `usePopularGuides(5)`, `useModuleStats()`
- **블록**:
  - Hero 배너 · 검색 CTA
  - 모듈 카드 6개 (MODULE_TREE) · 각 카드에 guide count badge
  - 최근 업데이트 리스트 (최대 8개)
  - 인기 가이드 리스트 (최대 5개)
- **권한**: 전체 공개. `canAccess` 에 해당하는 모듈만 카드 노출.

### 2.2 가이드 목록 `/guides` · `/modules/:moduleId`
- **데이터**: `useGuideList({ module, type, search })`
- **필터**: 모듈, 가이드 타입 (SCENARIO / PROCEDURE / POLICY / TROUBLESHOOT / FAQ / SCRIPT)
- **정렬**: 최신순 (updated_at desc) 기본, 조회수 / 도움됨률 옵션
- **페이지네이션**: 기본 20개 단위 (`usePagination` 훅)
- **엠티 상태**: 결과 0 → `NoResultFallback` (추천 가이드 + 가이드 요청 폼)

### 2.3 가이드 상세 `/guides/:id`
- **데이터**: `useGuide(id)` + 조회수 증가 (fire-and-forget)
- **섹션 (가이드 타입별 동적)**:
  - TL;DR (모든 타입 공통)
  - `steps` / `mainItemsTable` / `cases` / `cautions` / `troubleTable` / `responses` / `decisionTable` / `referenceData` / `policyDiff`
  - 연관 가이드 (같은 모듈·태그)
  - 피드백 (👍/👎 + 자유 코멘트 + 이메일)
- **액션**:
  - 복사 (링크, 가이드 본문 마크다운)
  - PDF 내보내기 (예정, V2)
  - Confluence 원본 바로가기 (있을 때)
- **권한**: VIEW 만 있으면 조회 가능. 편집 버튼은 `hasPermission('edit')` 시에만.

### 2.4 검색 (글로벌 오버레이, `⌘+K`)
- **트리거**: `Cmd/Ctrl + K` · 헤더 검색창 클릭
- **입력**: 300ms debounce
- **결과**: `searchGuides(q, 20)` + AI 요약 카드 (`useSearchSummary`)
- **AI 요약**:
  - `/api/search-summary` serverless → Claude `claude-opus-4-7`
  - `ANTHROPIC_API_KEY` 미설정 시 카드 자동 숨김
  - 질의 당 1회만 호출 (AbortController 로 직전 요청 취소)
- **엠티**: `NoResultFallback` (관련 가이드 제안 + 가이드 요청)

### 2.5 FAQ `/faq`
- `type === 'FAQ'` 인 가이드 필터 노출 · 아코디언 UI

### 2.6 업데이트 `/updates`
- 최근 2주 updated_at 오름차순. 모듈별 그룹.

### 2.7 피드백 `/feedback`
- 가이드 요청, 오타/오류 제보, 사용자 칭찬 3 탭
- 제출 → `guide_feedback` insert (공통 payload: `type`, `comment`, `email`, `session_id`)

## 3. 에디터 `/editor` · `/editor?id=:id`

### 3.1 목적
- 관리자·실장·상담실이 가이드를 **신규 작성 / 기존 수정 / 임시저장 / 발행** 할 수 있는 단일 툴

### 3.2 모드 판별
| URL | 모드 | 액션 |
|---|---|---|
| `/editor` | NEW | 빈 상태 → 최초 저장 시 `upsert` with new id |
| `/editor?id=:id` | EDIT | `fetchGuide(id)` 로 기존 내용 프리필 |

### 3.3 섹션 편집기
- **기본 메타**: 제목, 모듈, 타입, 대상자(targets), 태그, TL;DR, amsUrl, confluenceUrl
- **타입별 섹션** (`EMPTY_CONTENT[type]` 에 기반 초기값):
  - `steps` — 이미지·캡션·텍스트 블록 배열
  - `cautions` — 경고 카드 배열
  - `cases` — 케이스별 플로우 배열
  - `troubleTable` / `decisionTable` / `mainItemsTable` — 표 편집기
  - `responses` — 스크립트 배열
  - `policyDiff` — 변경 전/후 대비

### 3.4 자동저장
- `useAutosave(state, key, 1500ms)` 훅으로 로컬 초안 저장
- 복구: 에디터 진입 시 `localStorage` 에 초안 존재 확인 → "이전 편집 복구" 배너

### 3.5 발행 (publish)
- **Validation**: 제목 · 모듈 · 타입 · TL;DR · (최소 1개 섹션) 필수
- **Action**: `supabase.from('guides').upsert({ ..., status: 'published', updated_at: now() })`
- **Success**: Toast + `/guides/:id` 로 이동
- **Fallback (mock)**: localStorage 에 guide 객체 저장 (개발 전용)

### 3.6 임시저장 (draft)
- `status: 'draft'` 로 upsert
- 어드민 `/admin/guides` 에서 draft 탭으로 접근

### 3.7 권한
- **접근 가드**: `<RequireRole permission="edit">` 통과 필요
- **수정 권한 세분화**: counselor 는 `MODULE_ACCESS['counselor']` 에 속한 모듈만 편집

## 4. 어드민 (신규 구축)

### 4.1 공통 레이아웃
- **라우트 prefix**: `/admin/*`
- **`AdminLayout` 컴포넌트** (`src/layouts/AdminLayout.jsx`)
  - 좌측 `AdminSidebar` (app-sidebar 와 분리)
  - 상단 user avatar + "사이트로 돌아가기" 링크
  - Breadcrumb
- **가드**: `<RequireRole permission="manage_users" fallback="/403">` — 관리자만 전체 접근. 개별 페이지는 permission 별 세분화.

### 4.2 `/admin` — 대시보드
- **데이터**: `useDashboardStats()` + `useModuleStats()`
- **블록**:
  - KPI 카드 4개: 총 가이드 수 / 총 조회 / 도움됨률 / 검색 수
  - 모듈별 가이드 분포 (막대 목록 — recharts 제거됨)
  - 최근 피드백 5건 (guide_feedback order by created_at)
  - 최근 검색 쿼리 5건 (search_logs)
- **Empty**: 데이터 없음 → Placeholder 카드

### 4.3 `/admin/guides` — 가이드 관리
- **데이터**: `fetchAdminGuides({ status, module, q })` (신규 함수)
  - Supabase: `status in ('draft','published','archived')` 전부 포함
- **UI**:
  - 상태 필터 탭: 전체 / 발행됨 / 임시저장 / 보관됨
  - 모듈 드롭다운 · 제목 검색
  - 테이블: [제목] [모듈] [타입] [상태 배지] [작성자] [수정일] [액션]
- **액션 (DropdownMenu)**:
  - 편집 → `/editor?id=:id`
  - 발행/해제 → `status` 토글 upsert
  - 복제 → id 재발급 후 insert
  - 삭제 → 확인 다이얼로그 후 `delete().eq('id', id)` (soft delete 권장: `status='archived'`)
- **권한**: edit · publish · delete 를 버튼별로 분기

### 4.4 `/admin/feedback` — 피드백 수신함
- **데이터 소스 2개**:
  1. `guide_feedback` (Supabase) — 가이드 상세에서 온 👍/👎
  2. `localStorage.FEEDBACK_QUEUE_KEY` — NoResultFallback/FeedbackPage 에서 쌓인 요청(오프라인 안전망)
- **머지 규칙**: 로컬 큐 먼저 표시 + Supabase insert 성공 시 큐에서 제거
- **UI**:
  - 탭: 전체 / 가이드 요청 / 오류 제보 / 칭찬
  - 행: [타입 배지] [요약] [가이드 링크(있으면)] [이메일] [일시] [상태(대기/처리/반려)]
  - 액션: 담당자 배정, 상태 변경, 메모, 해당 요청으로 에디터 열기

### 4.5 `/admin/users` (V2 이월)
- 사용자 목록 · 역할 변경 · 초대 발송

### 4.6 `/admin/sync` (V2 이월)

### 5.2 신규 테이블 (Round 4 예정)

```sql
-- 사용자 프로필 (RBAC 서버화)
profiles (
  id uuid primary key references auth.users(id),
  name text, avatar_url text,
  role text not null default 'operator' check (role in ('admin','director','counselor','operator')),
  modules text[] default '{}',  -- COUNSELOR·OPERATOR 담당 모듈
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- 가이드 요청함
guide_requests (
  id uuid primary key default gen_random_uuid(),
  type text not null check (type in ('request','bug','praise')),
  title text, description text,
  module text, guide_id text references guides(id),
  email text, session_id text,
  status text default 'pending' check (status in ('pending','in_progress','resolved','rejected')),
  assignee uuid references profiles(id),
  note text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Confluence 동기화 실행 이력
confluence_sync_runs (
  id uuid primary key default gen_random_uuid(),
  started_at timestamptz default now(),
  finished_at timestamptz,
  status text check (status in ('running','success','failed')),
  pages_fetched int, pages_updated int, errors jsonb,
  actor uuid references profiles(id)
);
```

## 6. API (Vercel Serverless)

### 6.1 `GET /api/confluence-img/*`
- **용도**: Atlassian 이미지 CDN 인증 프록시
- **Auth**: `CONFLUENCE_EMAIL:CONFLUENCE_TOKEN` Basic (서버 env 전용)
- **제거**: 구 `VITE_CONFLUENCE_*` fallback → **P0 로 삭제 예정**

### 6.2 `POST /api/search-summary`
- **용도**: 검색 결과 AI 요약 카드
- **Input** (json): `{ query: string, results: Array<{title,tldr}> }`
- **Size limit**: 8KB (미구현 → 추가 예정)
- **Auth**: 서버 `ANTHROPIC_API_KEY`
- **Output**: `{ summary: string, suggestedQueries: string[] }`
- **Error**: 키 미설정 시 503 → 클라이언트에서 카드 숨김

## 9. 수용 기준 (Acceptance Criteria)

### 9.1 Round 4 (어드민 MVP)
- [ ] 비로그인 사용자가 `/editor`, `/admin`, `/admin/*` URL 직접 입력 시 로그인/403 으로 리다이렉트
